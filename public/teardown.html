The product in one line

Spin up a cloud browser for your agent in ~300ms — no infra to run.

Kernel gives AI agents a sandboxed Chromium they can drive via Playwright, Puppeteer, CDP or "computer use." Sessions persist (cookies/auth), you get live view + MP4 replay, stealth mode handles CAPTCHAs & proxies, and you can deploy the whole agent on Kernel. YC-backed, $22M Series A, open source.

The verdict

Doing well vs. missing

What's working
  • Speed as a story. "300ms / fastest infra" is a sharp, ownable wedge.
  • AI-native by default. llms.txt, paste-into-Cursor onboarding, agent templates.
  • Real moat features. Persistence, stealth, live replay, deploy-the-agent.
  • No proxy upcharge. A concrete, quotable trust signal.
  • Credibility. YC + $22M + open source = instant legitimacy.
What's missing
  • Friction at hello. Login before you can feel the product. Stripe lets you touch it first.
  • Landing page is thin. JS-only, little copy — no instant "what/why/proof" for a skimming dev.
  • Pricing is hard to model. Per-second rates don't map to "what will my agent cost me?"
  • Shallow examples. Few copy-paste, end-to-end recipes per framework.
  • Thin proof & narrative. Logos, benchmarks, and use-case stories under-told.
If I ran marketing — first 90 days

Five moves, highest leverage first

Each one closes a specific gap above and is measurable.

1

Kill the signup wall

An in-browser playground or anonymous CLI sandbox — drive a real browser before login, Stripe-style. Friction at hello is the single biggest leak.

Impact: top-of-funnel activation ↑
2

Rewrite the landing page around speed + proof

Above the fold: the 300ms claim, a live demo GIF, the 5-line code block, customer logos, and the "vs Browserbase/Steel" benchmark. Make it skimmable in 10 seconds.

Impact: bounce ↓, intent ↑
3

Ship a pricing calculator

"1,000 agent runs × 2 min = $X/mo." Translate per-second rates into outcomes. Reframe "no proxy fees" as money saved vs. Steel/Browserbase.

Impact: conversion confidence ↑
4

Build a recipe library

Copy-paste, end-to-end guides: "Agent that books a flight," "Scrape behind login," one per framework (Browser Use, Stagehand, Computer Use). This is where Steel currently out-docs Kernel.

Impact: time-to-value ↓
5

Lead with benchmarks & stories

Turn "fastest" from a claim into a reproducible benchmark page + 2–3 customer case studies. Make speed provable, not asserted.

Impact: trust & differentiation ↑
The one idea to steal from Stripe

Stripe didn't win on features. It won by letting a developer succeed in 60 seconds, before committing anything.

Kernel already has the 60-second magic moment (a live browser). The job is to move it earlier — before the login, before the install — and surround it with proof. Same product, dramatically better funnel.
